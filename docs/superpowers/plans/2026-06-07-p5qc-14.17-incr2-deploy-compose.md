# 14.17 Increment 2 — Deploy Compose (preserve + markerless host-local overlay)

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make `setforge install` deploy a `preserve_user_sections` markdown file (claude_md) so that every tracked-authored host-local marker pair is stripped from the deployed file and each migrated section's body is injected **once**, markerless, from its local.yaml OVERLAY span — with zero marker duplication and zero body leak.

**Architecture:** Thread the already-passed `spans`/`span_states` into `copy_atomic`'s ELSE (preserve) branch (disposition=None). After `_compute_content` renders the merged+hash-stamped content, blanket-strip every host-local marker pair (removes tracked placeholders AND empty-dropped regions) then `inject_overlay_bodies` the host-local OVERLAY spans, returning `new_span_states` for the sidecar. The double-injection trap (eb070aa projects overlay spans into `host_local_sections`, which `inject_all` would inject *with markers*) is closed by filtering overlay-anchor names out of the `host_local_sections` map before it reaches `_compute_content`/`inject_all`.

**Tech Stack:** Python 3.12, attrs/dataclasses, pytest, existing `setforge.deploy` / `setforge.sections` / `setforge.overlay_deploy` / `setforge.source` modules.

---

## In plain terms

Today, host-local sections in `CLAUDE.md` live as `<!-- ... -->` marker pairs that
`setforge install` writes into the deployed file. We're moving those bodies into
`local.yaml` as markerless "overlay" spans, so the deployed file ends up **clean —
no markers** — with each section's text spliced in invisibly.

Increment 2 is just the **deploy half**: when install renders the file, strip out
the leftover marker placeholders and splice each body in once from its overlay
span. (The other half — actually moving the bodies into `local.yaml` the first
time — is increment 3, deliberately not built here.)

The one thing that can go wrong: a recent fix made the loader hand the deploy code
the *same* section twice — once as an old-style "inject with markers" entry and
once as a new overlay span. If we're not careful it gets written **twice**. This
plan closes that with two cheap, belt-and-suspenders moves: don't feed the
overlay-named entries to the marker-injector, and blanket-strip every host-local
marker before splicing the clean bodies back in.

**The payoff, concretely:** after install, `~/.claude/CLAUDE.md` has zero
`setforge:user-section` markers, each populated section appears exactly once, the
two empty placeholder regions vanish, and your per-host edits survive re-install.

## The three seams

1. **`setforge/cli/_install_helpers.py:617-622`** — `file_spans`/`span_states` are
   loaded ONLY for `disposition is not None`. A preserve file (claude_md) with
   host-local OVERLAY spans must load them too, else `spans=None` reaches
   `copy_atomic` and the ELSE branch has nothing to inject. Writeback gate at
   `:657 if file_spans:` then fires correctly once `file_spans` is populated.

2. **`setforge/deploy.py:292-306`** — the ELSE branch calls `_compute_content`
   and ignores `spans`. Add: filter `host_local_sections` to legacy-only
   (drop names that are OVERLAY-span anchors), then after `_compute_content`,
   when host-local overlay spans exist, `strip_host_local_markers(content)` +
   `inject_overlay_bodies(content, md_overlay_spans, span_states)` → set
   `new_span_states` (already threaded into `_write_resolved_content:319`).

3. **No projection change** — `source._host_local_sections_for_overlay` and its
   4 consumers (capture strip / compare mask / promote / install inject) stay
   untouched; the legacy-only narrowing happens locally in `copy_atomic`.

## Double-injection trap (the load-bearing correctness decision)

Post-migration, `host_local_sections_map[claude_md]` is the **projection** —
it contains the migrated overlay sections (via the `SpanKind.OVERLAY` loop in
`_host_local_sections_for_overlay`). If that map reaches `inject_all`
(deploy.py:615) the section is injected WITH markers; `inject_overlay_bodies`
then injects it again markerless → duplicated. Fix: in `copy_atomic`'s ELSE
branch, build `legacy_host_local = {name: sec for name, sec in
host_local_sections if name not in overlay_anchor_names}` and pass THAT to
`_compute_content`. For claude_md this is `{}` (no legacy block), so `inject_all`
is a no-op; the blanket strip removes the 6 tracked placeholder pairs; the 4
populated overlay bodies inject markerless; the 2 empty regions have no span and
their placeholders are removed by the same blanket strip (= the spec's "drop
empty regions"). **Blanket** strip (not named) is required precisely because the
empty-dropped regions have no overlay span to name.

**Scope boundary (documented, not a bug):** a single preserve file mixing a
*legacy markered* host-local section AND a 14.17 overlay span is out of scope —
the blanket strip after `inject_all` would erase the legacy markered section.
The target files (claude_md) carry no legacy host-local block, so this cannot
occur. If a future file needs both, switch to a named strip scoped to overlay
anchors + the empty-region names; file a follow-up bd then.

```python
def _legacy_only_host_local(
    host_local_sections: dict[HostLocalSectionName, HostLocalSection] | None,
    spans: list[SpanEntry] | None,
) -> dict[HostLocalSectionName, HostLocalSection] | None:
    """Drop host-local entries whose name is a host-local OVERLAY span anchor.

    The loader projects migrated OVERLAY spans back INTO the host-local map
    (source._host_local_sections_for_overlay) for capture/compare/promote.
    On the deploy preserve path those names are injected MARKERLESS via
    inject_overlay_bodies, so they must NOT also reach inject_all (which would
    inject them WITH markers — double injection). Returns the map unchanged
    when there are no overlay spans (legacy-only files are byte-for-byte
    untouched).
    """
    if not host_local_sections or not spans:
        return host_local_sections
    overlay_names = {s.anchor for s in overlay_deploy.overlay_spans(spans)}
    if not overlay_names:
        return host_local_sections
    return {
        name: sec
        for name, sec in host_local_sections.items()
        if name not in overlay_names
    }
```
